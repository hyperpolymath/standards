== What is "Standard"

The Rhodium Standard Repository (RSR) v1.0 is the complete, frozen definition of what constitutes a compliant repository under the *Campaign for Cooler Coding and Programming (CCCP)*.

=== Canonical Definition

A *Rhodium Standard Repository* is one that meets the following immutable criteria:

[quote, RSR v1.0]
____
A Rhodium Standard Repository is a comprehensive framework for *emotionally safe*, *technically excellent*, and *politically autonomous* software development that considers the entire lifecycle from upstream dependencies to downstream human impact.
____

=== 3. Language Policy (LOCKED)

==== Allowed Languages

[cols="1,2"]
|===
| Language | Primary Use Case

| *ReScript* | Primary application code (OCaml soundness)
| *Rust* | Systems programming, WASM, CLI tools
| *Deno* | Runtime & package management (replaces Node)
| *Gleam* | Backend services (BEAM or JS target)
| *OCaml* | Compilers, formal methods
| *Ada* | Safety-critical systems (SPARK verification)
| *Julia* | Data processing, batch scripts
| *Guile Scheme* | State/meta files (STATE.scm, etc.)
| *Nickel* | Configuration language
| *Bash/POSIX* | Minimal scripting, automation
|===

==== Banned Languages

[cols="1,1,2"]
|===
| Language | Replacement | Rationale

| TypeScript | ReScript | Unsound gradual typing
| Node.js | Deno | Insecure by default
| npm/Bun/pnpm/yarn | Deno | Supply chain risks
| Go | Rust | Error handling, generics
| Python (general) | ReScript/Rust | No static types
| Java/Kotlin | Rust | JVM overhead
| Swift | Tauri/Dioxus | Platform lock-in
|===

*Exception*: Python permitted for SaltStack only (temporary, slated for removal).

== Versioning Policy

=== Semantic Versioning

RSR follows Semantic Versioning for the specification itself:

* *MAJOR* (X.0.0): Breaking changes, locked criteria modifications
* *MINOR* (1.X.0): New optional criteria, clarifications, addenda
* *PATCH* (1.0.X): Typo fixes, editorial corrections

=== Stability Guarantees

[cols="1,3"]
|===
| Guarantee | Description

| *Backward Compatibility*
| v1.0-compliant repos remain compliant in v1.x

| *Deprecation Notice*
| Criteria removal requires 2 minor versions notice

| *Freeze Commitment*
| Locked items cannot change without major version
|===

== [1.0.0] — 2025-12-27

=== Status

*FROZEN* — This is the first stable release of the RSR specification.

=== Added

* *Core Specification* (link:RSR-SPEC.adoc[RSR-SPEC.adoc])
** 6 core CCCP principles defined
** 4 compliance tiers (Bronze, Silver, Gold, Rhodium)
** 11 compliance categories with weighted scoring

* *Tier Definitions* (link:TIERS.adoc[TIERS.adoc])
** Bronze: 75-89% compliance
** Silver: 90-99% compliance
** Gold: 100% compliance
** Rhodium: 100% + exemplary practices

* *Language Policy* (link:LANGUAGE-POLICY.adoc[LANGUAGE-POLICY.adoc])
** 10 allowed languages defined
** 9 banned languages with replacements
** SaltStack Python exception documented

* *Machine-Readable Specs* (`spec.scm/`)
** `compliance-criteria.scm`: 50+ criteria across 11 categories
** `tiers.scm`: Tier definitions and scoring
** `language-policy.scm`: Language allowed/banned lists

* *Version Freeze* (link:VERSION.adoc[VERSION.adoc])
** Formal v1.0 freeze declaration
** Versioning policy established
** Stability guarantees defined

=== Locked

The following are now immutable in v1.x:

* Compliance tier score thresholds
* Compliance category weights
* Language allowed/banned lists
* Required file specifications
* Core CCCP principles
* TPCF contribution model

=== Satellites

The following satellite specifications are referenced:

* Palimpsest License v0.8+
* Mustfile (type-safe deployment)
* .well-known Ecosystem (RFC 5785)
* DocGementer (documentation validation)

== Pre-1.0 History

Prior to v1.0, RSR existed as an evolving set of best practices without formal versioning. The v1.0 freeze crystallizes the specification from work done throughout 2024-2025.

Key milestones:

* *2024-Q3*: Initial CCCP manifesto drafted
* *2024-Q4*: Tier system introduced
* *2025-Q1*: Language policy formalized
* *2025-Q2*: Satellite specifications added
* *2025-Q3*: Machine-readable specs (Guile Scheme)
* *2025-Q4*: v1.0 frozen

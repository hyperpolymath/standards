== Overview

The *Rhodium Standard Repository (RSR)* framework provides comprehensive standards for emotionally safe, technically excellent, and politically autonomous software development. It embodies the principles of the *Campaign for Cooler Coding and Programming (CCCP)*.

=== Why "Rhodium"?

Rhodium is the *rarest and most precious* of all metals, chosen for specific symbolic properties:

* **Rarest precious metal**: Excellence over ubiquity
* **Catalytic role**: Transforms toxic outputs into safer alternatives
* **Wedding ring plating**: Durability for long-term commitments and maintainability
* **Corrosion resistance**: Systems that don't decay under pressure

=== Core Principles

[cols="1,3",options="header"]
|===
| Principle | Description

| Emotional Safety
| Reversibility, safe experimentation, no shame in mistakes

| Offline-First
| Work continues without constant connectivity

| Post-JavaScript Liberation
| Rejecting npm/node ecosystem fragility

| Formal Verification
| Correctness as mutual aid

| Community Over Ego
| Architecture enforces collaborative patterns

| Language Polyglotism
| Using Ada, Chapel, Elixir, Haskell, Julia, ReScript, Rust instead of JavaScript/Python monoculture
|===

=== AI Context Files

RSR repositories include three machine-readable Scheme files for AI assistant context preservation:

[cols="1,3",options="header"]

|===

| File | Purpose

| `META.scm`

| Architecture decisions (ADRs), development practices, design rationale

| `ECOSYSTEM.scm`

| Project ecosystem position, relationships, explicit boundaries

| `STATE.scm`

| Current project state, progress tracking, session continuity

|===

These files work alongside `.claude/CLAUDE.md` (or similar AI instruction files) to provide structured context that persists across AI conversation sessions. See link:docs/ai-context-files.md[AI Context Files Specification] for complete documentation and links to the specification repositories.

== Features

=== RSR Audit System

Comprehensive compliance checking with 100+ automated tests:

[source,bash]
----
# Text output (default)
./rsr-audit.sh /path/to/repo

# JSON output (for CI/CD)
./rsr-audit.sh /path/to/repo json

# HTML report
./rsr-audit.sh /path/to/repo html > report.html
----

==== Exit Codes

* `0` - Gold compliance (100%)
* `1` - Silver compliance (90-99%)
* `2` - Bronze compliance (75-89%)
* `3` - Non-compliant (<75%)
* `4` - Error running audit

== RSR Compliance Categories

The RSR framework validates 11 comprehensive categories:

. **Foundational Infrastructure**: Nix, Justfile, GitLab CI, Podman, Git configuration
. **Documentation Standards**: README, LICENSE, SECURITY, CODE_OF_CONDUCT, CONTRIBUTING, .well-known/
. **Security Architecture**: Type safety, memory safety, SPDX headers, supply chain security
. **Architecture Principles**: Offline-first, CRDTs, reversibility, reproducibility
. **Web Standards & Protocols**: DNS, TLS, HTTP security headers (if applicable)
. **Semantic Web & IndieWeb**: Schema.org, RDF, Webmention (if applicable)
. **FOSS & Licensing**: License clarity, contributor rights, funding transparency
. **Cognitive Ergonomics**: Information architecture, accessibility, i18n
. **Lifecycle Management**: Dependency management, SemVer, EOL planning
. **Community & Governance**: TPCF framework, CoC, governance model
. **Mutually Assured Accountability**: MAA principles, audit trails, provenance

See link:COMPLIANCE_CHECKLIST.md[Compliance Checklist] for detailed requirements.

== Usage

=== Auditing Your Repository

[source,bash]
----
# Audit current directory
./rsr-audit.sh .

# Audit specific repository
./rsr-audit.sh /path/to/repo

# Get JSON output for CI/CD
./rsr-audit.sh . json | jq '.score'
----

=== Creating a New RSR-Compliant Repository

[source,bash]
----
# Copy template
cp -r templates/rust-cli my-new-project
cd my-new-project

# Initialize
just init

# Customize
$EDITOR README.adoc LICENSE.txt SECURITY.md

# Validate
just validate
----

=== Integrating with CI/CD

.GitLab CI Example
[source,yaml]
----
rsr-compliance:
  stage: validate
  script:
    - wget https://gitlab.com/hyperpolymath/rhodium-standard-repositories/-/raw/main/rsr-audit.sh
    - chmod +x rsr-audit.sh
    - ./rsr-audit.sh . json > audit.json
    - cat audit.json
  artifacts:
    reports:
      codequality: audit.json
    paths:
      - audit.json
  rules:
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'
----

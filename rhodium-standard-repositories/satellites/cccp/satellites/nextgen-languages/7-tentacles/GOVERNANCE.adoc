= Seven Tentacles Governance
:toc:

== Overview

Seven Tentacles is governed with the goal of creating an inclusive, educational
project that serves learners ages 8-18 worldwide.

== Project Leadership

=== Maintainers

The project is maintained by:

* **Jonathan D.A. Jewell** - Creator and Lead Maintainer

=== Decision Making

Decisions are made through:

1. **Consensus:** For most changes, we seek community consensus
2. **Maintainer Decision:** For urgent or architectural decisions
3. **Community Vote:** For significant directional changes

== Contribution Acceptance

=== Criteria for Acceptance

Pull requests are accepted based on:

* Alignment with project goals
* Code quality and testing
* Documentation completeness
* Accessibility compliance
* Age-appropriateness of content

=== Review Process

1. Automated checks must pass
2. At least one maintainer review
3. Community feedback period (for significant changes)
4. Merge by maintainer

== Emotionally Safe Development

We embrace shame-free experimentation:

=== Reversible Changes

* All changes should be reversible
* We prefer feature flags over big-bang releases
* Breaking changes require clear migration paths

=== Safe to Fail

* Failed experiments are learning opportunities
* We document what didn't work and why
* No blame for good-faith attempts that don't succeed

=== Inclusive Feedback

* Feedback focuses on code, not people
* We assume good intentions
* Criticism is constructive and specific

== Roadmap Process

=== Planning

* Quarterly planning discussions (open to community)
* Roadmap published in repository
* Community input welcomed

=== Priorities

1. **Accessibility:** Making the platform usable by all learners
2. **Content Quality:** High-quality, pedagogically sound lessons
3. **Technical Excellence:** Reliable, maintainable codebase
4. **Community Growth:** Building an engaged contributor base

== Conflict Resolution

If conflicts arise:

1. Discuss in the relevant issue/PR
2. Escalate to maintainers if needed
3. Final decision by lead maintainer
4. Appeals process for significant disagreements

== Amendments

This governance document may be amended through:

1. Proposal via pull request
2. Community discussion period (14 days)
3. Maintainer approval
4. Version update

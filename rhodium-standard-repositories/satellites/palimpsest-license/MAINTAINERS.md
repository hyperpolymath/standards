## Maintainer Responsibilities

### Core Duties
1. **Code Review**: Review and merge pull requests following TPCF perimeter policies
2. **Issue Triage**: Respond to issues within 7 days, prioritise appropriately
3. **Security**: Monitor security.txt reports, coordinate vulnerability disclosure
4. **Governance**: Participate in Stewardship Council votes when applicable
5. **Documentation**: Keep documentation current and accurate
6. **Community**: Enforce Code of Practice, foster inclusive environment

### Decision-Making Authority

| Decision Type | Authority | Process |
|---------------|-----------|---------|
| Bug fixes, typos | Any maintainer | Direct merge after review |
| Documentation improvements | Documentation maintainer | Direct merge |
| New features, enhancements | Lead maintainer + 1 other | 2-maintainer approval |
| License text changes | Stewardship Council | Formal governance process (see GOVERNANCE.md) |
| Major versions (v1.0, v2.0) | Stewardship Council | 6/7 vote, 90-day community review |
| Minor versions (v0.4, v0.5) | Stewardship Council | 5/7 vote, 30-day community review |

## Becoming a Maintainer

### Contributor â†’ Maintainer Path

1. **Contribute**: Make meaningful contributions (code, docs, community support)
2. **Demonstrate**: Show consistent engagement over 6+ months
3. **Nomination**: Existing maintainer nominates you
4. **Review**: Stewardship Council reviews contribution history
5. **Vote**: Simple majority (4/7) Council approval
6. **Onboarding**: Receive repository access, GPG key registration, documentation

### Expected Contributions
- **Code**: 10+ merged PRs or equivalent technical contributions
- **Documentation**: Significant guide improvements or new content
- **Community**: Active issue triage, helpful responses, mentoring
- **Governance**: Participation in discussions, proposal drafting

### Maintainer Criteria
- **Technical competence**: Understand the license framework and tools
- **Cultural sensitivity**: Respect for emotional lineage and cultural context
- **Communication**: Clear, respectful, professional
- **Commitment**: 4-8 hours/month minimum availability
- **Alignment**: Support the Palimpsest mission and values

## Maintainer Rotation & Retirement

### Active Status Requirements
- **Minimum activity**: 1 substantial contribution per quarter
- **Responsiveness**: Respond to mentions within 14 days
- **Availability**: Communicate planned absences >1 month

### Emeritus Status
Maintainers who step down in good standing become **Emeritus Maintainers**:
- Recognised in project credits
- Advisory role (no voting rights)
- Can return to active status by Council vote

### Removal Process
Inactive or problematic maintainers may be removed by:
1. **Voluntary**: Maintainer requests emeritus status
2. **Inactivity**: >6 months without contributions or responses
3. **Code of Practice violation**: Serious or repeated violations
4. **Council vote**: 5/7 Stewardship Council vote required
